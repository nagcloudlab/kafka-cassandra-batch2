Designing Microservice Persistence

  • Polyglot Persistence
    • Each service can use the most appropriate database technology.
    • Table 7-1: Polyglot persistence example.
    ServiceData characteristicsDatabase options
    Hotel ServiceDescriptive text about hotels and their amenities, changes infrequentlyDocument database (i.e., MongoDB), Cassandra, or Elasticsearch/Solr for full text search
    Point of Interest ServiceGeographic locations and descriptions of points of interestCassandra or other tabular databases supporting geospatial indexes such as DataStax Enterprise
    Inventory ServiceCounts of available rooms by date, large volume of reads and writesCassandra or other tabular databases
    Reservation ServiceRooms reserved on behalf of guests, lower volume of reads and writes than inventoryCassandra or other tabular databases
    Guest ServiceGuest identity and contact information, possible extension point for customer and fraud analytics systemsCassandra, graph databases
  • Representing Other Database Models in CQL
    • Key-value models: Treat key as partition key, store value as text/blob.
    • Document models: Use JSON for flexible schema.
    • Example: Creating a table with a JSON document.
      
CREATE TABLE hotel.hotels_document (
    id text PRIMARY KEY,
    document text);
    • Insert and select JSON data.
      
INSERT INTO hotels JSON '{ "id": "AZ123", "name": "Super Hotel Suites at WestWorld", "phone": "1-888-999-9999", "address": { "street": "10332 E. Bucking Bronco", "city": "Scottsdale", "state_or_province": "AZ", "postal_code": "85255" }}';
SELECT JSON * FROM hotels WHERE id = 'AZ123';
    • Graph models: Use a dedicated table for each vertex and edge type.

Alternatives to Denormalized Tables

  • Secondary Indexes
    • Create indexes for non-primary key columns.
    • Example: Creating an index on the hotel name column.
      
CREATE INDEX ON hotels (name);
    • Figure 7-8: Revised hotel physical model.
  • Materialized Views
    • Preconfigured views to support queries.
    • Example: Creating a materialized view for reservations by confirmation number.
      sqlCopy code
CREATE MATERIALIZED VIEW reservation.reservations_by_confirmation AS SELECT * FROM reservation.reservations_by_hotel_date WHERE confirm_number IS NOT NULL and hotel_id IS NOT NULL and start_date IS NOT NULL and room_number IS NOT NULL PRIMARY KEY (confirm_number, hotel_id, start_date, room_number);
