Cassandra Cluster Manager

  • Overview
    • Cassandra works out of the box with default settings but offers extensive configurability.
    • Cassandra Cluster Manager (ccm) is a toolset of Python scripts to manage multinode clusters on a single machine.
    • Ideal for testing and learning configuration options without additional hardware.
  • Installing ccm
    bashCopy code
$ pip install ccm
$ brew install ccm  # for macOS users
  • Creating a Cluster with ccm
    • Primary configuration file: cassandra.yaml.
    • Key values: cluster name, partitioner, snitch, seed nodes.
    bashCopy code
$ ccm create -v 4.0.0 -n 3 reservation_cluster --vnodes
$ ccm list
*reservation_cluster
$ ccm status
$ ccm start
  • Inspecting Cluster Configuration
    • Nodes and their configuration can be checked using ccm status and ccm showlog.
    bashCopy code
$ ccm node1 status
$ ccm node1 ring
$ cd ~/.ccm/reservation_cluster
$ diff node1/conf/ node2/conf/
  • Adding Nodes to a Cluster
    • Simple with ccm.
    bashCopy code
$ ccm add node4 -i 127.0.0.4 -j 7400
$ ccm start

Dynamic Ring Participation

  • Node Operations
    • Nodes can be stopped and started without disrupting the cluster.
    bashCopy code
$ ccm stop
$ ccm start

Node Configuration

  • Seed Nodes
    • Used as contact points for bootstrapping.
    • Example seed configuration in cassandra.yaml:
    yamlCopy code
- seeds: "127.0.0.1, 127.0.0.2, 127.0.0.3"
  • Snitches
    • Determine network topology.
    • Common snitches: SimpleSnitch, PropertyFileSnitch, GossipingPropertyFileSnitch, RackInferringSnitch.
    yamlCopy code
endpoint_snitch: GossipingPropertyFileSnitch

Partitioners

  • Default Partitioner
    • Murmur3Partitioner for even distribution of data.
    • Other options include RandomPartitioner and OrderPreservingPartitioner.
    yamlCopy code
partitioner: org.apache.cassandra.dht.Murmur3Partitioner

Tokens and Virtual Nodes

  • Configuration
    • Use vnodes for ease of management.
    yamlCopy code
num_tokens: 256
  • Disabling vnodes
    • Manually configure token ranges if not using vnodes.
    yamlCopy code
num_tokens: 1

Network Interfaces

  • Configuration Options
    yamlCopy code
listen_address: 127.0.0.1
broadcast_address: 127.0.0.1
rpc_address: 127.0.0.1
broadcast_rpc_address: 127.0.0.1
native_transport_port: 9042

Data Storage

  • Directory Configuration
    • Separate directories for datafiles, commit logs, hints, and caches.
    yamlCopy code
commitlog_directory: /var/lib/cassandra/commitlog
data_file_directories:
  - /var/lib/cassandra/data

Startup and JVM Settings

  • Configuration Files
    • jvm.options for JVM settings.
    • logback.xml for logging configuration.

Creating a Cluster in Docker

  • Docker Commands
    • Example of creating a cluster with Docker.
    bashCopy code
$ docker network create my-network
$ docker run --name node1 --network my-network -p 9042:9042 -d cassandra
$ docker run --name node2 --network my-network -e CASSANDRA_SEEDS=node1 -d cassandra
$ docker run --name node3 --network my-network -e CASSANDRA_SEEDS=node1,node2 -d cassandra

Planning a Cluster Deployment

  • Cluster Topology and Replication Strategies
    • Consider data center layout, replication needs, and workload isolation.
    • Example replication strategy for multiple data centers:
    sqlCopy code
ALTER KEYSPACE reservation
WITH REPLICATION = {'class' : 'NetworkTopologyStrategy', 'DC1' : '3', 'DC2' : '3'};
  • Sizing Your Cluster
    • Calculate storage size based on table schema, replication factor, and compaction strategy.
    • Recommended to balance storage and performance.
  • Selecting Instances
    • Development: 2 cores, 8 GB memory.
    • Production: 8+ cores, 32+ GB memory.
    • SSDs preferred for performance.

Cloud Deployment

  • Amazon Web Services (AWS)
    • Use EC2 instances with ephemeral or EBS storage.
    • VPC for networking.
    • Use Ec2Snitch or Ec2MultiRegionSnitch.
  • Google Cloud Platform (GCP)
    • Use GCE instances with local SSDs or network-attached storage.
    • Global VPC network recommended.
    • Use GoogleCloudSnitch.
  • Microsoft Azure
    • Use D series or H series VMs with Premium SSDs.
    • VNet peering for networking.
    • Use GossipingPropertyFileSnitch.
