Installing the Apache Distribution

  1. Download: Go to the Cassandra website (http://cassandra.apache.org) and download the latest version as a gzipped archive.
  1. Extract:
    • Use a ZIP utility to extract the tarball.
    • On Unix-based systems, use preinstalled gzip extraction utilities.
    • On Windows, use WinZip or 7-Zip.
  1. Directory Structure:
    • bin: Contains executables to run Cassandra and clients, including cqlsh.
    • conf: Configuration files for Cassandra.
    • doc: Documentation, including a getting started guide.
    • javadoc: JavaDoc-generated documentation.
    • lib: External libraries required by Cassandra.
    • pylib: Python libraries used by cqlsh.
    • tools: Maintenance tools for Cassandra nodes.

Building from Source

  1. Dependencies:
    • Apache Ant and Maven for build scripting and dependency management.
    • Java 8 JDK or later.
  1. Download Ant: From http://ant.apache.org.
  1. Clone Repository:
    • $ git clone https://github.com/apache/cassandra.git
  1. Build:
    • Navigate to the source root directory and run $ ant.
    • Ant and Maven will handle dependencies and build the project.

Running Cassandra

  1. Set JAVA_HOME: Configure your system environment to include the path to your Java installation.
  1. Start the Server:
    • Navigate to <cassandra-directory>/bin.
    • Run cassandra -f to start the server in the foreground.
  1. Configuration Logs: Review logs to ensure the server starts correctly.

Stopping Cassandra

  1. Stop Command:
    • Use the stop-server script in the bin directory.
    • Alternatively, use pgrep and kill to terminate the process.

Other Cassandra Distributions

  1. DataStax Enterprise Edition: Provides production support and integrated data technologies.
  1. Virtual Machine Images: Available in AWS Marketplace.
  1. Containers: Running Cassandra in Docker.
  1. Managed Services: Providers include Instaclustr, Aiven, and DataStax Astra.

Running the CQL Shell (cqlsh)

  1. Start cqlsh:
    • Navigate to the Cassandra home directory.
    • Run bin/cqlsh to connect to the local node.
  1. Basic Commands:
    • DESCRIBE CLUSTER: Show cluster information.
    • DESCRIBE KEYSPACES: List keyspaces.
    • SHOW VERSION: Display version information.

Creating and Managing Keyspaces and Tables

  1. Create Keyspace:
    • Example: CREATE KEYSPACE my_keyspace WITH replication = {'class': 'SimpleStrategy', 'replication_factor': 1};
  1. Switch Keyspace: USE my_keyspace;
  1. Create Table:
    • Example: CREATE TABLE user (first_name text, last_name text, title text, PRIMARY KEY (last_name, first_name));
  1. Insert Data:
    • Example: INSERT INTO user (first_name, last_name, title) VALUES ('Bill', 'Nguyen', 'Mr.');
  1. Query Data:
    • Example: SELECT * FROM user WHERE first_name='Bill' AND last_name='Nguyen';
  1. Delete Data:
    • Example: DELETE FROM user WHERE first_name='Bill' AND last_name='Nguyen';

Running Cassandra in Docker

  1. Pull and Run Cassandra:
    • $ docker pull cassandra
    • $ docker run --name my-cassandra cassandra
  1. Access cqlsh in Docker:
    • $ docker exec -it my-cassandra cqlsh
  1. Expose Ports:
    • $ docker start cassandra -p 9042:9042
