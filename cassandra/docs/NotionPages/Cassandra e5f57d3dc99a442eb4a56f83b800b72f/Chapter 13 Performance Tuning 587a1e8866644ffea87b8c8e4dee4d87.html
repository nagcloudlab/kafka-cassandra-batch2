Managing Performance

  • Continuous Process: Managing performance should start from application architecture and continue through development and operations.

Setting Performance Goals

  • Clear Goals: Set clear goals for throughput (number of queries per unit time) and latency (time to complete a query).
  • Understand Usage: Know the number of clients, usage patterns, and peak periods.
  • Example Goal: 30,000 read operations per second with 99th percentile read latency of 5 ms.

The USE Method

  • Utilization, Saturation, Errors: Track performance metrics for CPU, memory, disk, and network I/O.
  • Root Causes: Identify and address root causes of performance issues holistically.

Benchmarking and Stress Testing

  • Benchmarking: Measure performance under defined loads (e.g., Yahoo Cloud Serving Benchmark - YCSB).
  • Stress Testing: Increase load to discover errors and performance degradations (e.g., cassandra-stress tool).
  • Production Loads: Use cassandra-stress to create benchmarks and stress tests that resemble production loads.

Additional Load Testing Tools

  • tlp-stress: Command-line tool with simpler syntax and built-in workloads for common data models.
  • NoSQLBench: Extensible framework for running performance tests, provided by DataStax.
  • Apache JMeter: Open-source Java framework for functional and load testing.

Monitoring Performance

  • nodetool Commands: Use commands like tpstats, tablestats, proxyhistograms, and tablehistograms.
  • Trends and Issues: Identify performance trends and issues.
  • Automated Alerts: Implement automated alerts for key performance metrics.

Analyzing Performance Issues

  • Large Partitions: Detect large partitions by searching logs for WARN messages.
  • Anti-patterns: Recognize anti-patterns such as queues or large numbers of tombstones.
  • Tracing: Use tracing in cqlsh and DataStax drivers for detailed insights into query performance.

Tuning Methodology

  • Single Parameter Changes: Change one configuration parameter at a time and test results.
  • Monitor and Adjust: Monitor and adjust key metrics to maintain performance goals.

Caching

  • Key Cache: Maps partition keys to row index entries, speeding up read access.
  • Row Cache: Caches entire rows, useful for read-heavy workloads.
  • Chunk Cache: Stores decompressed chunks of SSTable data.
  • Counter Cache: Improves counter performance.
  • Balanced Configuration: Configure caches carefully to balance memory usage and performance gains.

Memtables

  • Memory Configuration: Configure heap and off-heap memory for memtables.
  • Flush Writers: Adjust memtable_flush_writers and enable metered flushing for predictable write I/O.

Commit Logs

  • Size and Compression: Set commit log size and compression for write-heavy workloads.
  • Sync Options: Configure commit log sync options to balance durability and performance.

SSTables

  • Separate Files: Separate commit logs and datafiles for performance.
  • Compression and Length: Configure SSTable compression, chunk length, index summaries, and Bloom filters.

Hinted Handoff

  • Disk and Bandwidth Tuning: Tune disk space and bandwidth utilization for hints.
  • Enable/Disable: Enable and disable hinted handoff as needed.

Compaction

  • Compaction Strategy: Choose the right compaction strategy (SizeTiered, Leveled, TimeWindow).
  • Thresholds and Status: Configure compaction thresholds and monitor compaction status.
  • Write Survey Mode: Use write survey mode to test compaction strategies.

Concurrency and Threading

  • Configure Reads/Writes: Configure concurrent reads and writes.
  • Task Management: Manage threads for various tasks (hint delivery, memtable flush, compaction).

Networking and Timeouts

  • Timeouts: Set timeouts for various operations.
  • Bandwidth and Connections: Configure bandwidth throttles and limit client connections.

JVM Settings

  • Heap Size and GC: Tune JVM heap size and garbage collection.
  • Different GC Options: Use different garbage collectors (CMS, G1GC, ZGC, Shenandoah).

Operating System Tuning

  • Linux OS Settings: Apply recommended Linux OS settings for Cassandra.
  • Monitoring Tools: Use performance monitoring tools for troubleshooting.

Tuning Cassandra Clients

  • Optimizations: Use prepared statements, compression, request tracking, and throttling in DataStax Java Driver.

Summary

  • Plan, Monitor, Debug: Plan, monitor, and debug performance using stress tools and performance metrics.
