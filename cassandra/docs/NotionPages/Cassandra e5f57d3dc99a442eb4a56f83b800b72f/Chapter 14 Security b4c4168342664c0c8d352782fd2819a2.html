Chapter 14. Security

Introduction

  • Data Accessibility vs. Security: The tension between making data accessible for analytics and meeting growing security and privacy demands.
  • Regulatory Compliance: Increased regulations like GDPR, CCPA, HIPAA, PCI DSS, and Sarbanes-Oxley impact how systems must secure data.
  • NoSQL Security: Despite historical criticisms, Cassandra has significantly improved its security features, including authentication, authorization, encryption, and audit logging.

Authentication and Authorization

Password Authenticator

  • Default Configuration: Cassandra allows any client to connect by default using AllowAllAuthenticator.
  • Using PasswordAuthenticator:
    • Change authenticator in cassandra.yaml to PasswordAuthenticator.
    • Restart the cluster and log in using the default user cassandra with password cassandra.
    • Change the default password for the cassandra user.

Adding Users

  • Create Users: Use CREATE USER <username> WITH PASSWORD '<password>'.
  • Superuser Role: Only superusers can create other users and manage permissions.
  • Example:
    sqlCopy code
CREATE USER reservation_service WITH PASSWORD 'password';
ALTER USER cassandra WITH PASSWORD 'new_password';
  • Automate Login: Store credentials in .cqlshrc.

Cassandra Authorizer

  • Enable Authorization: Change authorizer in cassandra.yaml to CassandraAuthorizer.
  • Grant Permissions: Use GRANT <permission> ON <resource> TO <user>.
    sqlCopy code
GRANT SELECT ON KEYSPACE reservation TO reservation_service;
GRANT MODIFY ON KEYSPACE reservation TO reservation_service;

Role-Based Access Control (RBAC)

  • Create Roles: Assign permissions to roles and grant roles to users.
    sqlCopy code
CREATE ROLE reservation_maintenance;
GRANT ALL ON KEYSPACE reservation TO reservation_maintenance;
GRANT reservation_maintenance TO jeff;

Encryption

SSL, TLS, and Certificates

  • Generate Certificates: Use keytool to create public/private key pairs.
  • Configure Keystores and Truststores: Export public keys and import them into truststores.
  • Example Commands:
    shCopy code
keytool -genkey -keyalg RSA -alias node1 -keystore node1.keystore -storepass password -keypass password
keytool -export -alias node1 -file node1.cer -keystore node1.keystore
keytool -import -v -trustcacerts -alias node1 -file node1.cer -keystore node2.truststore

Node-to-Node Encryption

  • Enable Encryption: Configure server_encryption_options in cassandra.yaml.
    yamlCopy code
server_encryption_options:
    enabled: true
    keystore: conf/.keystore
    keystore_password: password
    truststore: conf/.truststore
    truststore_password: password
    internode_encryption: all

Client-to-Node Encryption

  • Configure Client Encryption: Set client_encryption_options in cassandra.yaml.
    yamlCopy code
client_encryption_options:
    enabled: true
    keystore: conf/.keystore
    keystore_password: password
    truststore: conf/.truststore
    truststore_password: password

JMX Security

  • Enable Remote JMX Access: Edit cassandra-env.sh to set LOCAL_JMX=no.
  • Configure Authentication: Set JMX authentication properties.
    shCopy code
JVM_OPTS="$JVM_OPTS -Dcom.sun.management.jmxremote.authenticate=true"
JVM_OPTS="$JVM_OPTS -Dcom.sun.management.jmxremote.password.file=/etc/cassandra/jmxremote.password"
  • Secure Password and Access Files: Configure jmxremote.password and jmxremote.access.

Audit Logging

  • Enable Audit Logging: Configure audit_logging_options in cassandra.yaml.
    yamlCopy code
audit_logging_options:
    enabled: true
    logger: BinAuditLogger
    included_keyspaces: reservation
    included_categories: QUERY,DML
  • Log Example:
    shCopy code
INFO  [Native-Transport-Requests-1] ... user:reservation_service|host:127.0.0.2:7000|...

Summary

  • Security Configuration: Secure Cassandra using authentication, authorization, encryption, and audit logging.
  • Role Management: Use roles for efficient permission management.
  • Data Encryption: Protect data in motion between clients and nodes.
  • JMX Security: Secure remote JMX access.
  • Compliance: Ensure compliance through audit logging.
